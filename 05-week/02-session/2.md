### Ejercicio 2

A partir de las tablas **`user_account`**, **`user_role`** y **`role`**, construya una **consulta SQL** que permita obtener la **lista de roles que ya están asignados a usuarios**.

El objetivo **no es listar todos los roles existentes en la tabla `role`**, sino únicamente aquellos que **ya tienen usuarios asociados**.

**Condiciones obligatorias:**

- Usar **INNER JOIN**
- Aplicar **`DISTINCT`**
- Mostrar únicamente:
  - `r.name AS list_role`
- Alias requeridos:
  - `u` → `user_account` | `ur` → `user_role` | `r` → `role`
- Relaciones entre tablas:
  - `u.id = ur.user_id` | `r.id = ur.role_id`

El resultado debe mostrar **cada rol una sola vez**, sin repetirse, aunque esté asignado a varios usuarios.

---

### Salida esperada (ejemplo)

| list_role |
|-----------|
| admin |
| invitado |
| editor |
